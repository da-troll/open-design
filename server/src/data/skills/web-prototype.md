# Skill: web-prototype

## Output
A complete, single-page HTML document (self-contained, no external dependencies except CDN fonts and optionally Tailwind CDN).

## Structure
- Full responsive layout
- Semantic HTML5
- All CSS inline in <style> tag or inline styles
- Interactive states (hover, focus) in CSS
- At most one JavaScript block for simple interactions (tabs, modals, toggles)
- Realistic placeholder content (no "Lorem Ipsum" — generate contextually appropriate copy)
- 4–8 sections typical: nav, hero, features/content, CTA, footer

## Image placeholders
Use `<img-gen prompt="[descriptive prompt matching the design context]">` for any meaningful images.
Use CSS background colors or gradients for decorative elements.

## Quality bar
Must look like a real, polished product page — not a wireframe. Apply the active design system fully.
